タイムラインパネルを使用すると、Gaussian Splatシーン用のカメラアニメーションを作成できます。キーフレームを設定してカメラの位置と動きを定義し、エディター内で再生できる滑らかなアニメーション遷移を作成できます。

![タイムラインパネル](/img/user-manual/supersplat/editor/timeline.png)

## タイムラインコントロール

タイムラインパネルには次のコントロールがあります：

- **再生/一時停止ボタン** - アニメーションの再生を開始または一時停止します
- **前/次のフレームボタン** - 1フレームずつ前後に移動します。**Shift**を押しながらクリックすると、前または次のキーフレームにジャンプします
- **キー追加ボタン (`+`)** - 現在のタイムライン位置に新しいキーフレームを作成します
- **キー削除ボタン (`-`)** - 現在の再生ヘッド位置のキーフレームを削除します
- **タイムラインスライダー** - 再生ヘッドをドラッグしてアニメーションをスクラブします（再生ヘッドには現在のフレーム番号が表示されます）
- **FPS設定** - アニメーションのフレームレートを設定します
- **タイムライン長** - アニメーションの総時間をフレーム単位で設定します
- **スムーズネス** - キーフレーム間の補間の滑らかさを調整します (0-1)
- **ループトグル** - カメラパスをループさせるかどうかを制御します。有効（デフォルト）の場合、アニメーションは最後のキーフレームから最初のキーフレームへ滑らかに循環します。無効の場合、カメラは最後のキーフレームで静止します。[アニメーションのループ](#アニメーションのループ)を参照してください

### キーボードショートカット

キーボードでもタイムラインを操作できます：

| キー | 動作 |
| --- | --- |
| **Space** | アニメーションの再生/一時停止 |
| `,` / `.` | 前/次のフレームに移動 |
| `<` / `>` | 前/次のキーフレームにジャンプ |
| **Enter** | 現在のフレームにキーフレームを追加 |
| **Shift + Enter** | 現在のフレームのキーフレームを削除 |
| **Ctrl + T**（Macでは**Cmd + T**） | タイムラインパネルの表示/非表示を切り替え |

## キーフレームの作成

カメラアニメーションのキーフレームを作成するには：

1. タイムラインの再生ヘッドを目的のフレームに配置します（タイムラインをクリックするか、フレームカウンターを使用）
2. ビューポートカメラを目的の位置に移動して向きを調整します
3. タイムラインの`+`（キー追加）ボタンをクリックします（または**Enter**キーを押します）

キーフレームは、現在のフレーム位置のタイムラインに黄色いダイヤモンドとして表示されます。キーフレームには、その時点でのカメラの位置と回転が保存されます。

## カメラポーズをキーフレームとしてインポート {#importing-camera-poses-as-keyframes}

SuperSplatは、[COLMAP](https://colmap.github.io/)再構築またはINRIAカメラポーズJSONファイルからカメラポーズをインポートして、タイムラインのキーフレームを自動的に作成できます。これにより、Gaussian Splatシーンの元の撮影時に使用されたおおよそのカメラパスを再現できます。

### COLMAP `images.txt`

COLMAPポーズをインポートするには：

1. COLMAPリコンストラクション出力から[`images.txt`](https://colmap.github.io/format.html#images-txt)ファイルを取得します
2. SuperSplatで**File > Import**を選択します（または`images.txt`ファイルをドラッグアンドドロップします）
3. SuperSplatはファイル内のカメラポーズからキーフレームを作成します

`images.txt`ファイル内の各カメラポーズは、ファイルに表示される順序で、タイムラインの連続するフレームにキーフレームとして配置されます。これにより、元のキャプチャパスに従うアニメーションが作成されます。

:::note

`images.txt`ファイルには、COLMAPの座標系でクォータニオンと並進ベクトルとして表現されたカメラポーズが含まれています。SuperSplatは、これらをタイムライン用の適切なカメラ位置と向きに自動的に変換します。

:::

### INRIAカメラポーズJSON

SuperSplatは、INRIAカメラエントリの配列を含む`.json`ファイルにも対応しています。各エントリには`position`と`rotation`が必要です。また、並び順を指定する`id`または`img_name`、視野角を計算する`fx`、`fy`、`width`、`height`を含めることができます。

**File > Import**を使用するか、ファイルをEditorへドラッグしてインポートします。SuperSplatは`id`、または`img_name`末尾の番号でエントリを並べ替え、各エントリからキーフレームを作成します。`meta.json`はアンバンドルSOGシーン用に予約されているため、別のファイル名を使用してください。

## キーフレームの編集

キーフレームを作成した後、次の方法で変更できます：

### キーフレームのタイミングを変更する

黄色いダイヤモンドのキーフレームをタイムライン上でクリックしてドラッグし、各カメラ位置がアニメーション内で発生するタイミングを調整します。

### キーフレームのコピー

**Shift**を押しながらキーフレームをドラッグすると、移動ではなくコピーになります。ドラッグ中はキーフレームのクローンがポインターに追従し、ドラッグを離したフレームにコピーが作成されます。元のキーフレームはそのまま残ります。

### カメラの位置/回転を更新する

既存のキーフレームに保存されているカメラの位置または回転を変更するには：

1. 更新したいキーフレームを含むフレームに再生ヘッドを移動します
2. ビューポートカメラを新しい目的の位置と回転に調整します
3. `+`（キー追加）ボタンをクリックします - これにより、既存のキーフレームが更新されたカメラ値で上書きされます

または、**Ctrl**を押しながらキーフレームのダイヤモンドをクリックすると、現在のビューで直接上書きできます - 再生ヘッドを先に移動する必要はありません。

## キーフレームの削除

キーフレームを削除するには：

1. 削除したいキーフレームを含むフレームに再生ヘッドを移動します
2. `-`（キー削除）ボタンをクリックします（または**Shift + Enter**を押します）

:::note

キーフレームの操作（追加、移動、コピー、削除）はすべて、**Ctrl + Z**（Macでは**Cmd + Z**）で元に戻し、**Ctrl + Shift + Z**（Macでは**Cmd + Shift + Z**）でやり直すことができます。

:::

## キーフレームカメラの可視化

キーフレームがカメラを3D空間のどこに配置するかを確認するには、**設定**パネル（[右ツールバー](interface.md#右ツールバー)の歯車アイコン）で**カメラを表示**トグルを有効にします。各キーフレームに保存された位置と向きに、ワイヤーフレームのカメラギズモがビューポートに描画されるため、カメラパスをひと目で確認できます。このトグルの状態はプロジェクトと共に保存されます。

## タイムライン設定の構成

### フレームレート (FPS)

FPS設定は、アニメーションの再生速度を制御します。利用可能なFPS値は次のとおりです：

- **1 FPS** - 非常に遅い、1秒あたり1フレーム
- **6 FPS** - 低フレームレート
- **12 FPS** - 中程度のフレームレート
- **24 FPS** - 標準的な映画のフレームレート
- **30 FPS** - 一般的なビデオフレームレート
- **60 FPS** - 滑らかで高いフレームレート再生

### タイムライン長

タイムライン長の設定を調整することで、アニメーションの総時間を設定できます。これにより、アニメーションが何フレームにわたるかが定義され、アニメーションが最初に戻るタイミングが決まります。

既存のキーフレームが終端より後になるようにタイムラインを短くした場合、SuperSplatはそれらのキーフレームを保持しますが、再生と補間から除外します。淡いエラー色のマーカーがタイムラインの右端のすぐ外側に固定され、ツールチップに各キーフレームの元のフレーム番号が表示されます。

範囲外のキーフレームをアニメーションに戻すには、固定されたマーカーをタイムライン上にドラッグするか、そのキーフレームが再び範囲内になるまで**Total Frames**を増やします。タイムラインを短くすると、再生ヘッドも新しい終端に収まり、前/次のキーフレームへの移動では範囲外のキーがスキップされます。範囲外のキーフレームを削除するには、まず範囲内にドラッグするか、**Total Frames**を一時的に増やしてください。

### スムーズネス

スムーズネス設定は、キーフレーム間でカメラがどのように補間されるかを制御します。この値の範囲は0から1です：

- **0** - 線形補間、キーフレーム間の直線的な動き
- **1** - 最大限の滑らかな補間、より曲線的で映画的なカメラの動き
- **中間の値** - 線形補間と滑らかな補間の間をブレンド

スムーズネスを調整することで、機械的で正確なものから、流れるような有機的なものまで、カメラアニメーションの感触を制御できます。

## アニメーションの再生

再生ボタン（または**Space**キー）を使用してアニメーションをプレビューします。カメラはスムーズネス設定に基づいてキーフレーム間を補間し、流動的な動きを作成します。ループトグルが有効（デフォルト）の場合、アニメーションは最後のキーフレームから最初のキーフレームへ循環しながら継続的にループします。

### アニメーションのループ

タイムラインのループトグル（スムーズネス設定の隣）は、カメラが最後のキーフレームを通過した後の動作を制御します：

- **ループ有効（デフォルト）** - カメラパスは閉じたループになります：最後のキーフレームの後、カメラはタイムラインの残りのフレームをかけて最初のキーフレームへ滑らかに補間されます
- **ループ無効** - カメラパスは開いたままになります：カメラは最後のキーフレームで静止し、そのポーズを保持します

ループトグルの状態はプロジェクトと共に保存され、**カメラを表示**トグルで描画されるカメラパスも、閉じた曲線または開いた曲線として更新されます。

ループが有効な場合、タイムラインの最初と最後のフレームの両方にキーフレームを設定しないことが重要です。これを行うと、アニメーションがループする際に突然のスナップが発生します。

:::tip[スムーズなループのベストプラクティス]

最初と最後のフレームの両方にキーフレームを配置するのではなく、最後のフレームの前に最後のキーフレームを配置し、アニメーションを最初に滑らかに補間させます。

例えば、100フレームのタイムラインの場合：

- **良い例:** フレーム0、10、20、30、40、50、60、70、80、90にキーフレームを設定
  - アニメーションはフレーム90からフレーム0まで滑らかに補間されます
- **避けるべき:** フレーム0、10、20、...90、100にキーフレームを設定
  - これにより、フレーム100からフレーム0にループする際にスナップ/ジャークが発生します

このテクニックにより、カメラは最後のキーフレームから開始位置まで滑らかに遷移し、シームレスなループを作成できます。

:::

## シーンページでの公開と再生

タイムラインアニメーションはSuperSplatプロジェクトに保存されます。訪問者に再生させるには、公開ダイアログの**Animation**トグルを有効にしてスプラットを[公開](/user-manual/supersplat/editor/publishing)します。これにより、キーフレーム化されたカメラパスが公開シーンにベイクされ、読み込み時に再生されるよう設定されます。ダイアログの**Loop Mode**オプションはタイムラインのループトグルから初期化され（ループ有効なら**Repeat**、無効なら**None**）、公開されたアニメーションをビューアで1回だけ再生するかループさせるかを制御します。

[シーンページ](/user-manual/supersplat/scene-page)では、オープンソースの[SuperSplat Viewer](/user-manual/supersplat/viewer/)が、訪問者がシーンを開いたときにトラックを自動再生します。トラックのループモードに応じて、一度再生して停止するか、連続してループします — `repeat`ループにすると、スプラットを手放しのシネマティックなフライスルーに変えられます。

:::note

カメラアニメーショントラックは、ここエディターのタイムラインで作成します — [SuperSplat Studio](/user-manual/supersplat/studio/)では作成できません。Studioはカメラ、注釈、ポストエフェクト、スカイボックス、コリジョンのみを調整します。公開設定でアニメーションがどのように保存されるか — `animTracks`配列（各トラックの`loopMode`、`interpolation`、キーフレーム）と、シーンレベルの`startMode` — については、[Experience Settings](/user-manual/supersplat/studio/experience-settings)を参照してください。

:::

## タイムラインアニメーションの保存

:::important

タイムラインアニメーションは、プロジェクトを`.ssproj`ファイルとして保存した場合にのみ保持されます。タイムライン設定は、PLYやその他のスプラット形式にエクスポートする際には保存されません。

SuperSplatプロジェクトファイルの保存と読み込みの詳細については、[プロジェクトの管理](managing-projects.md)を参照してください。

:::
